## 基于 GPU 批量并行传播的复杂网络节点传播桥梁中心性计算方法及系统

---

## 一、发明名称

**基于 GPU 批量并行传播的复杂网络节点传播桥梁中心性计算方法及系统**

---

## 二、技术领域

本发明涉及复杂网络分析、传播动力学建模与高性能并行计算技术领域，具体涉及一种面向流行病传播、信息扩散和风险扩散等一对多传播场景的节点传播桥梁中心性计算方法及系统。该方法基于平均场 SIR 传播动力学构建节点传播贡献度量，并利用图形处理器（GPU）的批量并行消息传递与反向贡献归因机制实现加速计算。

## 三、背景技术

### 3.1 现有中心性方法的基本范式

复杂网络中节点重要性分析通常依赖桥梁中心性等指标。其中，最常见的一类方法以最短路径搜索、广度优先遍历或路径采样为基础，通过统计节点位于多少条路径上来刻画其“中介”作用。当前常见的实现方式包括：

- 以精确最短路径遍历为基础的标准最短路径桥梁中心性计算；
- 以随机节点对采样和最短路径抽样为基础的近似方法；
- 以自适应采样或分层采样为基础的加速近似方法。

在本申请对应代码中，`exact_bc`、`BP_sampling_bc`、`RK`、`ABRA` 以及 `SILVAN.run_fast()` 均属于上述路径搜索型或路径采样型对照方法。

### 3.2 现有路径搜索型中心性计算存在的技术问题

现有桥梁中心性计算通常需要从大量源节点出发反复执行图遍历，并统计节点在路径中的中介贡献。即便采用采样近似、并行 BFS 或 GPU 图遍历等加速策略，相关计算过程仍容易受到以下因素限制：

- 图邻接访问不规则，内存访问局部性较差；
- 多数方法仍需要按源节点逐一展开；
- 节点贡献依赖路径结构统计，过程难以规整化；
- 不同源节点和不同区域子图的计算负载不均衡；
- 难以自然组织为规则、连续的批量张量计算流程。

因此，在大规模网络中，这类方法往往面临运行时间增长快、GPU 利用率受限以及工程实现复杂等问题。

### 3.3 一对多传播场景中的度量需求

在流行病传播、信息扩散和风险扩散等场景中，节点的重要性并不完全取决于其是否处于最短路径上，而更取决于其在多源传播过程中的状态演化，以及其对后续扩散造成的直接或间接影响。

若仍沿用路径搜索型中心性框架，不仅难以准确刻画一对多扩散过程中的动态影响，也难以充分发挥 GPU 在批量状态更新、向量化消息计算和并行归约方面的计算优势。

因此，有必要从路径搜索型中心性计算范式转向传播动力学型中心性计算范式，以传播过程本身作为基本计算单元，将多源传播状态演化组织为 GPU 友好的批量并行计算，并进一步通过贡献归因机制刻画节点在传播过程中的桥梁作用。

## 四、发明内容

### 4.1 发明目的

本发明旨在提供一种基于平均场 SIR 传播动力学的节点传播桥梁中心性计算方法及系统，用于在复杂网络中刻画节点在传播过程中的桥梁作用，并通过 GPU 批量并行消息传递和反向贡献归因机制降低 CPU/Python 层面的逐源循环与邻居遍历开销。

### 4.2 发明的核心定位

本发明的主体方法并非最短路径桥梁中心性（SPBC）的近似算法，也不是 SPBC 的 GPU 版本。本发明定义的是一种**传播动力学桥梁中心性**，其核心思想是：

- 将节点桥梁性从“位于多少条最短路径上”转化为“在传播过程中对后续扩散产生多少直接或间接贡献”；
- 使用平均场 SIR 状态演化描述传播过程；
- 使用 GPU 对多个传播源同时执行批量传播和反向归因。

在本申请对应实现中：

- `exact_bc`、`BP_sampling_bc`、`RK`、`ABRA`、`SILVAN.run_fast()` 为对照方法；
- `ParallelSIRBridgeCentrality` 和 `compute_gpu_bridge_centrality` 为本发明主体方法；
- `evaluate_method` 与 `run_experiment` 用于对主体方法与对照方法进行统一评估。

### 4.3 三层创新点

#### 第一层：平均场 SIR 传播动力学建模

本发明不再以最短路径计数为核心，而是基于传播状态转移构建节点传播桥梁贡献度量。对节点 \(j\) 在时刻 \(t\) 的传播状态，采用平均场 SIR 模型描述：

\[
q_j(t)=\prod_{i \in N(j)} \left(1-\beta_{ij} I_i(t)\right)
\]

\[
S_j(t+1)=S_j(t)\, q_j(t)
\]

\[
I_j(t+1)=(1-\gamma)I_j(t)+S_j(t)\left(1-q_j(t)\right)
\]

\[
R_j(t+1)=R_j(t)+\gamma I_j(t)
\]

其中，\(S_j(t)\)、\(I_j(t)\)、\(R_j(t)\) 分别表示节点 \(j\) 在时刻 \(t\) 处于易感、感染、恢复状态的概率或强度，\(\beta_{ij}\) 为边级传播参数，\(\gamma\) 为恢复参数。

#### 第二层：GPU 批量并行传播计算

本发明将多个源节点组织为 batch 维度，同时推进传播过程。图结构表示为：

- `edge_index=[src,dst]`；
- 可选 `edge_weight`；
- 节点状态张量采用批量形式存储。

传播过程中，先通过 gather 获取每条边源节点的感染状态，再将感染失败概率的乘积转化为对数空间求和，利用 `scatter_add` 对目标节点聚合消息，其核心表达为：

\[
\log q_j(t)=\sum_i \log\left(1-\beta_{ij}I_i(t)\right)
\]

随后通过指数运算恢复存活概率，并并行更新 `S/I/R` 状态。该过程天然适合 GPU 张量计算与并行归约。

#### 第三层：正向传播与反向归因闭环

本发明不仅执行正向传播，还保留各时间步状态历史，并从末时刻向前进行传播贡献回传：

- 正向阶段保存 `S_hist`、`I_hist`、`R_hist`、`new_inf_hist`；
- 反向阶段从 \(T-1\) 到 \(0\) 计算边对下游传播的贡献比例；
- 将节点对后续传播的直接与间接贡献逐步回传；
- 最终聚合所有源节点和所有时间步的贡献，得到节点传播桥梁中心性。

## 五、技术方案

### 5.1 输入与输出

本发明方法的输入包括：

- 复杂网络图 `G`；
- 可选边权字段 `weight_attr`；
- 传播概率 `beta`；
- 恢复概率 `gamma`；
- 传播步数 `steps`；
- 可选源节点集合 `sources`；
- 批大小 `batch_size`；
- 设备参数 `device`；
- 数值稳定参数 `eps`；
- 归一化选项 `normalize`；
- 提前终止参数 `early_stop` 与 `threshold`。

输出为每个节点的传播桥梁中心性分数，形式为：

```text
{node: score}
```

并可输出运行时间等评估信息。

### 5.2 图预处理

本发明使用 `nx_to_pyg_edge_index` 将 NetworkX 图转换为适于 GPU 处理的边列表结构，具体包括：

1. 读取图 `G` 的节点集合并建立连续编号映射 `node_map`；
2. 建立反向映射 `rev_map`，用于结果回写原始节点编号；
3. 将图中的边提取为 `edge_index`；
4. 若设置了 `weight_attr`，则同步生成 `edge_weight`；
5. 对无向图添加反向边；
6. 按需将 `edge_index` 和 `edge_weight` 移动到指定计算设备。

其中：

- `edge_index` 的 shape 为 `(2, E)`；
- `edge_weight` 的 shape 为 `(E,)` 或为空；
- `node_map` 和 `rev_map` 分别用于节点编号压缩与恢复。

### 5.3 源节点批量化

在 `compute_gpu_bridge_centrality` 中：

- 若 `sources=None`，则以全部节点作为传播源；
- 若指定 `sources`，则仅对指定源节点集合进行传播计算；
- 所有源节点按照 `batch_size` 分批送入 GPU，以控制显存使用。

该设计使得本方法既可用于全图节点传播桥梁中心性分析，也可用于部分候选源节点的局部分析。

### 5.4 正向传播

本发明使用 `ParallelSIRBridgeCentrality.forward()` 完成批量正向传播。

#### 步骤 F1：状态初始化

对于一个 batch 中的源节点集合：

- 初始化 `S` 为全 1；
- 初始化 `I` 为全 0；
- 初始化 `R` 为全 0；
- 对每个 batch 对应的源节点位置，将 `I[b, source_nodes[b]] = 1`；
- 同时令 `S[b, source_nodes[b]] = 0`。

#### 步骤 F2：构造边级传播参数

- 若存在 `edge_weight`，则边级传播参数为 `beta_e = beta * edge_weight`；
- 否则所有边共享统一传播参数 `beta`。

#### 步骤 F3：逐时间步传播更新

对每个时间步执行以下计算：

1. 获取每条边源节点的感染状态：

```text
I_src = I[:, src]
```

2. 计算边级感染失败概率并做数值截断：

```text
prob = clamp(1 - beta_e * I_src, eps, 1.0)
```

3. 转到对数空间：

```text
msg = log(prob)
```

4. 按目标节点聚合：

```text
log_q = scatter_add(msg, dst)
```

5. 取指数恢复存活概率：

```text
q = exp(log_q)
```

6. 更新感染新增量与 S/I/R 状态：

```text
new_inf = S * (1 - q)
S_new = S * q
I_new = (1 - gamma) * I + new_inf
R_new = R + gamma * I
```

#### 步骤 F4：传播历史缓存

在每个时间步，本发明保存：

- `S_hist[t]`
- `I_hist[t]`
- `R_hist[t]`
- `new_inf_hist[t]`

从而形成后续反向归因所需的完整传播历史。

### 5.5 反向贡献归因

本发明在正向传播完成后，初始化贡献张量 `C`，并从 `T-1` 反向迭代至 `0`。

#### 步骤 B1：计算边级贡献比例

对每一时刻 `t`，先计算：

```text
numerator = I_hist[t][:, src] * beta_e
denom = scatter_add(numerator, dst) + eps
delta = numerator / denom[:, dst]
```

其中：

- `numerator` 表示当前时刻边级传播贡献基数；
- `denom` 表示汇聚到每个目标节点的总传播基数；
- `delta` 表示每条边在目标节点传播中的相对贡献比例。

#### 步骤 B2：计算下游传播贡献

基于下一时刻累计贡献与下一时刻感染状态，计算：

```text
downstream = (delta + C[t+1][:, dst]) * I_hist[t+1][:, dst]
```

该式用于表示：当前边除了产生即时贡献外，还通过下游节点继续影响后续传播。

#### 步骤 B3：将贡献回传到源节点

通过 `scatter_add` 按源节点回聚：

```text
C[t] = scatter_add(delta * downstream, src)
```

从而得到每个时刻每个节点的归因贡献。

### 5.6 中心性输出

本发明对每个源节点的节点传播桥梁分数按时间维度聚合：

```text
score_per_source = (I_hist[:T+1] * C[:T+1]).sum(dim=0)
```

随后：

1. 对不同 batch 的 `score_per_source` 结果进行拼接；
2. 对全部源节点结果取平均；
3. 根据 `rev_map` 将内部连续编号恢复为原始节点编号；
4. 输出 `{node: score}` 形式的节点传播桥梁中心性结果。

### 5.7 实验评估模块

本发明对应代码还包含统一评估模块，用于将主体方法与多种对照方法进行比较。其主要流程包括：

- 以 `exact_bc(G)` 计算精确最短路径桥梁中心性，作为对照基准；
- 分别运行 `BP_sampling_bc`、`RK`、`ABRA`、`SILVAN.run_fast()` 和 `compute_gpu_bridge_centrality()`；
- 使用 `evaluate_method` 统一计算：
  - `MAE`
  - `Top-k Accuracy`
  - `Spearman` 排名相关系数
  - `runtime`
- 使用 `run_experiment` 将结果写入：
  - `result/time.csv`
  - `result/mae.csv`
  - `result/acc.csv`
  - `result/spearman.csv`

需要说明的是，上述评估模块用于比较传播动力学桥梁中心性方法与路径搜索型桥梁中心性方法之间的差异及运行效率，并不意味着本发明主体方法是 SPBC 的近似或替代实现。

### 5.8 可选实施方式说明

当前代码实现的主体方法为基于平均场 SIR 的批量传播与反向归因计算。对于更大规模网络，可在进一步实施例中引入 Monte Carlo 采样扩展，但该扩展并非当前代码的已实现主体流程。

## 六、有益效果

### 6.1 计算组织方式更适合 GPU

与依赖最短路径遍历、路径枚举或路径采样的对照方法相比，本发明将传播过程组织为规则的张量计算流程，包括：

- 边级 gather；
- 对数空间消息计算；
- 基于 `scatter_add` 的节点聚合；
- 批量状态更新；
- 批量反向归因。

该方式更适合 CUDA、MPS 以及其他 GPU 或并行张量后端执行。

### 6.2 能刻画传播过程中的桥梁作用

本发明不再仅以最短路径中的“经过次数”作为节点重要性依据，而是从传播过程中的状态演化与后续扩散贡献出发，对节点桥梁作用进行刻画。因此，本发明与 SPBC、BP、RK、ABRA、SILVAN 所反映的节点重要性机制不同。

### 6.3 支持批量化多源分析

通过将多个传播源同时组织为 batch 维度，本发明减少了 Python/CPU 层面的逐源循环和邻居遍历开销，有利于在多源传播分析中提升吞吐能力。

### 6.4 便于统一评估与实验复现

当前代码已经内置对照实验流程，可将主体方法与多种对照算法统一输出到结果文件中，便于采用 MAE、Top-k Accuracy、Spearman 排名相关系数和运行时间等指标进行比较。具体实验数值以实际运行生成的 `result/*.csv` 文件为准。

## 七、权利要求书（草案）

**权利要求 1**  
一种基于 GPU 批量并行传播的复杂网络节点传播桥梁中心性计算方法，其特征在于，包括以下步骤：  
S1：输入复杂网络图，并将所述复杂网络图转换为适于张量计算的 GPU 边列表表示；  
S2：建立原始节点编号到连续编号的映射，并生成边索引 `edge_index` 以及可选边权重向量；  
S3：将多个源节点组织为 batch 维度，初始化每个源节点对应的 S/I/R 传播状态；  
S4：基于平均场 SIR 传播模型，在每个时间步对所有 batch 中的源节点同时执行传播更新；  
S5：对边级感染失败概率采用对数空间变换，并通过 `scatter_add` 按目标节点执行并行消息聚合；  
S6：保存每个时间步的传播历史状态；  
S7：基于传播历史状态，从末时刻向前执行反向贡献归因，计算节点对后续传播的直接和间接贡献；  
S8：对各源节点和各时间步的贡献结果进行聚合，得到各节点的传播桥梁中心性分数。

**权利要求 2**  
根据权利要求 1 所述的方法，其特征在于，步骤 S5 中将传播失败概率的乘积运算转化为对数空间求和运算，即将 \(\prod(1-x)\) 转换为 \(\exp(\sum \log(1-x))\)。

**权利要求 3**  
根据权利要求 1 所述的方法，其特征在于，步骤 S5 中的消息聚合通过 `scatter_add` 按目标节点索引完成，以实现边级消息到节点级状态的并行归约。

**权利要求 4**  
根据权利要求 1 所述的方法，其特征在于，步骤 S3 中将多个源节点同时作为 batch 输入传播模型，以并行计算多个源节点下的传播过程。

**权利要求 5**  
根据权利要求 1 所述的方法，其特征在于，对源节点集合按 `batch_size` 分批处理，以降低显存占用并适配不同硬件资源。

**权利要求 6**  
根据权利要求 1 所述的方法，其特征在于，支持可选边权重输入，并将全局传播参数与边权重相结合形成边级传播参数。

**权利要求 7**  
根据权利要求 1 所述的方法，其特征在于，所述计算设备支持 CUDA、MPS 或 CPU 回退执行方式。

**权利要求 8**  
根据权利要求 1 所述的方法，其特征在于，在正向传播过程中保存 `S_hist`、`I_hist`、`R_hist` 以及 `new_inf_hist` 作为传播历史缓存，以供反向归因使用。

**权利要求 9**  
根据权利要求 1 所述的方法，其特征在于，步骤 S7 中的反向贡献归因包括：计算边级贡献比例 `delta`、计算下游传播贡献 `downstream`，并通过 `scatter_add` 将贡献回传至源节点。

**权利要求 10**  
根据权利要求 1 所述的方法，其特征在于，还包括实验评估步骤，用于将所述传播桥梁中心性结果与精确最短路径桥梁中心性或近似最短路径桥梁中心性方法进行比较，并输出 MAE、Top-k Accuracy、Spearman 排名相关系数和运行时间。

**权利要求 11**  
一种基于 GPU 批量并行传播的复杂网络节点传播桥梁中心性计算系统，其特征在于，包括：  
- 图预处理模块，用于将复杂网络图转换为边列表、边权重以及节点映射；  
- 批量传播引擎，用于基于平均场 SIR 模型对多个源节点执行批量传播；

## 九、实施例与实验说明

### 9.1 对照方法说明

为说明本发明主体方法与传统路径搜索型中心性方法的差异，可在相同图数据上运行以下对照方法：

- `exact_bc`：精确最短路径桥梁中心性；
- `BP_sampling_bc`：随机最短路径采样近似方法；
- `RK`：随机采样式最短路径桥梁中心性近似方法；
- `ABRA`：基于 `compute_f_w` 的自适应采样近似方法；
- `SILVAN.run_fast()`：简化采样估计方法；
- `compute_gpu_bridge_centrality`：本发明主体方法。

### 9.2 对比指标说明

当前代码中可直接输出以下对比指标：

- MAE；
- Top-k Accuracy；
- Spearman 排名相关系数；
- 运行时间。

对应结果文件为：

- `result/time.csv`
- `result/mae.csv`
- `result/acc.csv`
- `result/spearman.csv`

### 9.3 对比结论的表述方式

由于本发明主体方法刻画的是传播动力学桥梁作用，而对照方法大多刻画的是最短路径中的中介作用，因此二者识别出的关键节点可能不同。对比实验可以用于分析：

- 本发明主体方法与 SPBC 的排名相关性；
- 本发明主体方法与路径采样近似方法的差异；
- 不同方法在 Top-k 节点集合上的重叠情况；
- 不同方法的运行时间差异。

该类对比用于说明不同中心性机制捕捉到的节点重要性特征不同，而不应简单表述为某一种方法在所有场景下必然优于另一种方法。
